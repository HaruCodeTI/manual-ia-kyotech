Este documento reúne exemplos de perguntas e cenários de uso para guiar os
testes do sistema. O objetivo é cobrir os principais casos de uso que um
técnico de campo ou especialista técnico encontraria no dia a dia.

COMO USAR ESTE GUIA
-------------------
- Acesse o chat e digite as perguntas abaixo como se fosse um técnico real
- Observe se a resposta cita a fonte (manual ou informativo) e o número da página
- Clique nas citações [Fonte N] para visualizar o trecho original do documento
- Teste variações: reformule com suas próprias palavras e veja se o sistema entende

================================================================================
  BLOCO 11 — TESTES DE COMPORTAMENTO DO SISTEMA
================================================================================

Esses exemplos testam casos-limite para verificar como o sistema responde
quando não há informação suficiente ou a pergunta é ambígua.

TESTE DE EQUIPAMENTO NÃO ESPECIFICADO:
101. "Como trocar o fusor?" (sem mencionar o equipamento)
     → Esperado: sistema pede para especificar o equipamento OU responde
       com informações genéricas se tiver documentação aplicável a todos.

TESTE DE INFORMAÇÃO INEXISTENTE:
102. "Qual é o número de série desse equipamento?"
     → Esperado: sistema informa que não tem essa informação no manual.

TESTE DE PERGUNTA FORA DO ESCOPO:
103. "Qual é o preço de venda da Jet Press 750S?"
     → Esperado: sistema informa que não tem informações comerciais.

TESTE DE PERGUNTA AMBÍGUA:
104. "Como resolver o erro 23?"
     → Esperado: sistema pede mais contexto (qual equipamento?) ou retorna
       informações do código de erro mais relevante encontrado na base.

TESTE DE SINÔNIMO / ABREVIAÇÃO:
105. "Problema no JP750S" (usando abreviação)
     → Esperado: sistema reconhece JP750S como Jet Press 750S.

106. "Erro na 1120" (usando somente número do modelo)
     → Esperado: sistema identifica como Revoria PC1120.

TESTE DE PERGUNTA EM INGLÊS:
107. "How to clean the print head on Jet Press 750S?"
     → Esperado: sistema responde em português com as informações do manual.

================================================================================
  DICAS PARA UM BOM TESTE
================================================================================

  ✓ Teste tanto perguntas curtas quanto perguntas detalhadas
  ✓ Verifique sempre se as citações [Fonte N] aparecem nas respostas
  ✓ Clique nas fontes para confirmar que o trecho citado é relevante
  ✓ Tente reformular a mesma pergunta de forma diferente
  ✓ Inicie conversas novas para testar sem contexto anterior
  ✓ Experimente mencionar o equipamento de formas diferentes
    (ex: "Jet Press 750S", "JP750S", "750S", "a 750")
  ✓ Teste perguntas com terminologia técnica e também com linguagem simples
  ✓ Registre qualquer resposta incorreta ou incompleta para feedback
